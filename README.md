# КОДЕКС-НАВИГАТОР

Интеллектуальный RAG-ассистент по кодексам Российской Федерации для поиска, объяснения и навигации по правовым нормам на естественном языке.

## О проекте

**КОДЕКС-НАВИГАТОР** — это учебный проект, реализующий подход **Retrieval-Augmented Generation (RAG)** для работы с юридическими документами.  
Система принимает вопрос пользователя на русском языке, находит релевантные фрагменты кодексов РФ и формирует структурированный ответ на основе найденного контекста.

Проект ориентирован на:

- поиск статей кодексов по смыслу запроса;
- краткое и понятное объяснение правовых норм;
- цитирование источников из нормативной базы;
- безопасное использование LLM в юридическом домене;
- сбор продуктовых и технических метрик.

> **Важно:** система не заменяет профессиональную юридическую консультацию и не предназначена для выдачи персонализированных юридических рекомендаций.

---

## Цель проекта

Цель проекта — создать интеллектуального помощника, который помогает пользователю быстро ориентироваться в законодательстве РФ и получать ответы по кодексам в удобной форме через Telegram-интерфейс.

---

## Основные возможности

- обработка юридических документов и разбиение их на смысловые фрагменты;
- векторизация текстов и построение поиска по эмбеддингам;
- использование LLM для генерации ответа с опорой на найденный контекст;
- Telegram-бот для общения с пользователем;
- хранение истории запросов и обратной связи;
- мониторинг пользовательских и технических метрик через Dash.

## Архитектура решения

Проект состоит из нескольких логических модулей:

### 1. Подготовка данных

Модуль `chunking.py` отвечает за обработку исходного HTML-документа кодекса:

- загружает HTML-файл;
- извлекает текст;
- разбивает его на логические фрагменты;
- сохраняет полученные чанки в CSV-файл.

Результат этого этапа — структурированный набор текстовых фрагментов, пригодных для последующей векторизации.

### 2. Построение эмбеддингов и индекса

Модуль `embedding.py`:

- читает подготовленные чанки из CSV;
- строит эмбеддинги для каждого фрагмента;
- индексирует их в **FAISS**;
- сохраняет векторную базу локально.

На этом этапе формируется поисковый слой системы.

### 3. Генерация ответа

Модуль `generator.py` реализует основной класс генерации ответа.

Логика работы:

1. пользователь задаёт вопрос;
2. система ищет наиболее релевантные фрагменты в FAISS;
3. найденный контекст передаётся в языковую модель;
4. модель формирует ответ по системному промпту;
5. ответ проходит дополнительную проверку качества;
6. при необходимости возвращается безопасный fallback-ответ.

### 4. Telegram-бот

Модуль `llmbot.py` реализует Telegram-бота, который:

- принимает сообщения пользователя;
- передаёт вопрос в генератор;
- сохраняет запрос и ответ в базу данных;
- отправляет ответ обратно пользователю;
- собирает реакции пользователя на качество ответа.

### 5. Мониторинг

Папка `monitoring/` содержит Dash-приложение для анализа работы системы.

Поддерживаются два направления мониторинга:

- **пользовательский мониторинг** — количество запросов, популярные слова, распределение лайков и дизлайков;
- **технический мониторинг** — доля успешных ответов, fallback-ответы, latency, confidence, число попыток генерации.

### 6. Запуск сервиса

Файл `run_service.py` объединяет запуск:

- Telegram-бота;
- Dash-дашборда мониторинга.

## Как работает система

Ниже представлен полный сценарий работы проекта:

1. Исходный HTML-файл кодекса обрабатывается в `chunking.py`.
2. Полученные текстовые фрагменты сохраняются в CSV.
3. В `embedding.py` для них строятся эмбеддинги.
4. Эмбеддинги индексируются в FAISS.
5. Пользователь задаёт вопрос через Telegram-бот.
6. Система находит близкие по смыслу фрагменты нормативного текста.
7. LLM получает вопрос и контекст.
8. Генератор формирует ответ на русском языке.
9. Ответ проверяется по заданным ограничениям.
10. Результат сохраняется в базу и отправляется пользователю.
11. Метрики работы системы отображаются в Dash-интерфейсе.

## Используемый стек технологий

### Язык программирования

- **Python 3.11**

### Работа с LLM и RAG

- **Hugging Face Inference API** — доступ к языковой модели
- **Qwen/Qwen2.5-7B-Instruct** — генерация ответов
- **LangChain Community** — работа с векторным хранилищем
- **langchain-huggingface** — подключение эмбеддинг-моделей
- **FAISS** — быстрый поиск по векторам
- **Sentence Transformers / HuggingFace Embeddings** — построение эмбеддингов
- **deepvk/USER-base** — модель эмбеддингов

### Обработка данных

- **pandas** — работа с таблицами и CSV
- **BeautifulSoup** — парсинг HTML-документов
- **sqlite3** — хранение истории запросов и технических метрик

### Интерфейсы

- **aiogram** — Telegram-бот
- **Dash** — веб-интерфейс мониторинга
- **Plotly** — интерактивные графики
- **Matplotlib** — визуализация
- **WordCloud** — облако слов

### Конфигурация и инфраструктура

- **python-dotenv** — загрузка переменных окружения
- **threading / asyncio** — параллельный запуск сервисов
- **Amvera** — деплой приложения
